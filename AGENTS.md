# AGENTS.md

## 역할
이 에이전트는 `별꿈다락 굿즈 스토어` 레포에서 MVP를 빠르게 런칭하기 위한 코딩 작업을 수행한다.

## 프로젝트 개요
- 프로젝트명: 별꿈다락 굿즈 스토어 (e-commerce)
- 목표: MVP를 빠르게 런칭할 수 있는 쇼핑몰 웹앱 구현 및 검증

## 개발 원칙
- TypeScript 기반으로 구현한다.
- ESLint, Prettier를 기본 품질 게이트로 사용한다.
- UI는 모바일 우선으로 구현한다.
- 접근성(키보드 내비게이션, ARIA 속성)을 기본 요구사항으로 포함한다.
- 반응형 레이아웃을 기본 적용한다.
- 커밋은 기능 단위로 작게 나눈다.
- 환경변수는 `.env.example`에 반드시 문서화한다.
- API/DB는 교체 가능하도록 레이어를 분리한다.
- 민감정보(키, 토큰, 비밀번호 등)는 절대 하드코딩하지 않는다.

## Codex 작업 방식
모든 작업은 아래 순서를 따른다.
1. 계획 작성: 작업 시작 전 `steps`를 먼저 제시한다.
2. 실행: 계획 순서대로 구현/수정한다.
3. 변경 요약: 수정 파일과 핵심 변경사항을 요약한다.
4. 검증 제시: 실행한 테스트 결과 또는 로컬 테스트 명령을 제시한다.

## 테스트/검증 원칙
- 새 기능마다 최소한의 테스트를 추가한다.
- 테스트 추가가 어려운 경우, 재현 가능한 수동 검증 절차를 반드시 남긴다.
- 버그 수정 시 회귀 방지 관점의 검증 절차를 포함한다.

## 아키텍처 원칙
- 프레젠테이션(UI), 도메인(비즈니스), 인프라(API/DB) 레이어를 분리한다.
- 외부 API/DB 의존성은 인터페이스로 추상화해 교체 가능성을 유지한다.
- 컴포넌트/모듈은 단일 책임 원칙을 우선한다.

## 환경변수 규칙
- 새 환경변수가 필요하면 `.env.example`에 키, 설명, 예시값을 즉시 추가한다.
- 런타임에서 필수 환경변수 누락 시 명확한 에러를 반환한다.
- 실제 시크릿 값은 `.env` 또는 배포 환경 변수로만 주입한다.

## 주요 명령 (기본)
프로젝트 스크립트는 아래 명령을 기준으로 유지한다.
- 의존성 설치: `npm install`
- 개발 서버: `npm run dev`
- 프로덕션 빌드: `npm run build`
- 프로덕션 실행: `npm run start`
- 린트: `npm run lint`
- 타입체크: `npm run typecheck`
- 테스트: `npm run test`
- 포맷: `npm run format`

## 완료 보고 템플릿
작업 완료 시 아래 형식으로 보고한다.
- 계획(steps)
- 실행 내용
- 변경 파일 요약
- 테스트/검증 결과
- 로컬 실행 명령
